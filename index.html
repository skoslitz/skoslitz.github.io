Umgang mit Dateien

Benennung:

Dateinamen der *.md (Content markdown), *.jpg (Bilder) und weitere mehr, bitte ausschließlich in Kleinbuchstaben, ohne Sonderzeichen und Leerzeilen. Es gilt eine Ausnahme für die Id der Dateien. Diese sind in Großschreibung und mit Unterstrichen zu versehen.

Beispiele:

78_B_104-bauphasen-der-innnenstadt.md
78_E_504-titelfoto.jpg

Markdown Syntax

Überschriften

Es stehen fünf Überschriftsebenen zur Verfügung. H1 (Header 1) und H2 sind jedoch bereits durch das Seitenlayout belegt.

Somit stehen im Contentbereich der Beiträge nur die Überschriftsgrößen H3 - H5 zur Verfügung

Syntax: Ein Rautenzeichen # leitet eine Überschrift ein.

### Überschrift 3
#### Überschrift 4
##### Überschrift 5

Hervorhebungen

Es sind kursive und fette Hervorhebungen möglich.

Syntax kursiv: 

*kursiver Text*

Syntax fett: 

**fett geschriebener Text**

Listen

Listen dienen der besseren Ansicht von Aufzählungen. Es kann dabei zwischen unsortierten und sortierten Aufzählungen unterschieden werden.

Syntax unsortierte Liste:

  + Punkt 1
  + Punkt 2

Syntax sortierte Liste:

  1. Punkt 1
  2. Punkt 2
  3. Punkt 3
      + Unterpunkt zur Punkt 3

Links

Syntax Link:

[Bezeichnung des Links](Adresse des Links)

Beispiel:

[Wikipedia](https://www.wikipedia.org/)

Sonderzeichen

Sonderzeichen (wie z.B: łaziti) werden dank UTF-8 Kodierung korrekt in markdown dargestellt. Beinhaltet der Zeichensatz jedoch ein Markdownsteuerzeichen (z.B: ein *), so muß diesem zwingend ein Backslash \ vorangestellt werden

Syntax Sonderzeichen mit Markdownsteuerzeichen:

\*Sukołazy wird zu *Sukołazy

Sonderzeichen: Gedankenstrich

Ein Gedankenstrich wird in markdown in UTF-8 Kodierung dargestellt:

Syntax Gedankenstrich:

Windows: alt + 150

Mac OS: alt + -

Frontmatter (Metadaten des Beitrages)

Die Frontmatter Markup Sprache ist TOML. Alle Frontmatter Inhalte müssen somit valides TOML sein.

Beiträge verfassen

Um einen neuen Beitrag zu verfassen, stehen derzeit zwei Vorlagedateien zur Verfügung.

Themenvorlage: /lid-site/content/themen/78_B_001-themenvorlage.md

Exkursionsvorlage: /lid-site/content/exkursionen/78_E_001-exkursionsvorlage.md

Slideshow einbinden

Wenn der Beitrag eine Slideshow enthalten soll, so werden alle benötigten Informationen im Frontmatter angegeben

Syntax Frontmatter:

[[slideshow]]
  bild = "/img/78/beitrag/bild1.jpg"
  titel = "Erstes Bild"
  untertitel = "Untertitel Bild 1"
  
[[slideshow]]
  bild = "/img/78/beitrag/bild2.jpg"
  titel = "Zweites Bild"
  untertitel = "Untertitel Bild 2"

Video einbinden

Wenn der Beitrag ein oder mehrere Videos enthalten soll, so werden alle benötigten Informationen im Frontmatter angegeben.

Syntax Frontmatter:

[[video]]
  id = "Vimeo_id_des_Videos"
  titel = "Blühende Landschaften"
  untertitel = "Eine Reise durch ..."
  beschreibung = "Hier kommt die Beschreibung rein"
  quelle = "Wer hat's erfunden?"
  laenge = "55 Minuten"

Exkursion einbinden

Die Koordinaten der Suchpunkte und alle weiteren relevanten Informationen für die Exkursion werden im Frontmatter angegeben

Syntax Frontmatter:

[[exkursion]]
  lat = 53.1233
  lon = 12.3454
  
[[exkursion]]
  lat = 51.1233
  lon = 10.3454

Entwurf Kartenzentrum

Das Kartenzentrum (centroid) wird im Frontmatter angegeben:

Syntax Frontmatter:

centroid = [51.311758, 12.343027]

Entwurf Titel der Stationsbeschreibung

Die Überschrift der jeweiligen Stationsbeschreibung erfolgt im Contentbereich via Shortcode. Neben dem Titel der Station muss zwingend die Stationsnummer übergeben werden:

Syntax Shortcode:

{{% stationstitel titel="Die Lauer" nummer="1" %}}

Entwurf Literaturangaben

Literaturangaben der einzelnen Stationsbeschreibungen werden im Frontmatter angegeben:

Syntax Frontmatter:

literaturangaben = ["C. Montgomery Burns’ Wegweiser zur Weltherrschaft, 2014", "Homers letzter Satz, 2015"]

Features via shortcodes einbinden

Shortcodes dienen der Erweiterung von Markdown. Eingebunden werden diese mit {{% shortcode %}}.

Derzeit stehen folgende Features per Shortcode zur Verfügung:

Bild (Abbildung) einfügen:

Syntax Shortcode:

{{% bild pfad="" titel=""  quellenangaben="" %}}

Beispiel:

{{% bild pfad="/78/beitrag/78_B_101/abb01.jpg" titel="N.N." quellenangaben="N.N."  %}}
